# MBA 케이스 스터디 챗봇 시나리오 기획서

## 1. 제품 개요

### 1.1 제품명 (가칭)
**CaseForMe** - 나만의 경영 케이스 스터디 챗봇

### 1.2 핵심 컨셉
MBA 경영 케이스 스터디를 **사용자의 인적사항(업종, 직급, 경력, 관심 분야)에 맞게 자동 변환**하여 제공하고, 소크라테스식 대화를 통해 **의사결정 역량을 훈련**하는 AI 챗봇

### 1.3 타겟 사용자
- 기업 교육 대상자 (중간관리자, 팀장급)
- MBA 준비생 또는 재학생
- 경영 역량 향상을 원하는 실무자
- 기업 내부 리더십 프로그램 참가자

### 1.4 핵심 가치
| 기존 케이스 스터디의 한계 | CaseForMe의 해결책 |
|------------------------|-------------------|
| 외국 기업 사례라 공감이 어려움 | 사용자의 업종/규모에 맞게 자동 변환 |
| 혼자 학습하면 토론 기회 없음 | AI가 소크라테스식 질문으로 토론 파트너 역할 |
| 정해진 케이스만 반복 | 사용자 맥락에 따라 무한한 변형 생성 |
| 피드백 없음 | AI가 의사결정 분석 리포트 제공 |

## 2. 사용자 흐름 (User Flow)

### 전체 흐름도
```
[1. 온보딩]     →  [2. 케이스 선택]  →  [3. 케이스 변환/제시]
     ↓                    ↓                      ↓
프로필 입력          카테고리 선택           맞춤형 케이스 읽기
                    난이도 선택
                                                 ↓
[6. 결과 리포트]  ←  [5. 의사결정]   ←  [4. 소크라테스 토론]
     ↓                    ↓                      ↓
분석 리포트           최종 결정 제출         질문-답변 대화
역량 피드백           근거 제시              관점 확장
```

## 3. 단계별 상세 시나리오

### Phase 1: 온보딩 - 사용자 프로필 입력

#### 수집 정보
```
필수 입력:
├── 이름
├── 현재 직무/직급 (예: 마케팅팀 과장)
├── 소속 산업/업종 (예: IT/소프트웨어, 제조업, 유통업 등)
├── 회사 규모 (스타트업/중소/중견/대기업)
└── 경력 연수

선택 입력:
├── 관심 경영 영역 (전략/마케팅/재무/HR/운영 등 복수 선택)
├── 최근 고민하는 업무 이슈 (자유 기술)
└── 학습 목표 (의사결정력/리더십/분석력 등)
```

#### UI 설계
- **대화형 온보딩**: 챗봇이 한 항목씩 자연스럽게 질문
- **진행률 표시**: 프로필 완성도를 프로그레스 바로 표시
- **건너뛰기 가능**: 선택 항목은 스킵 가능

#### 챗봇 대화 예시
```
🤖: 안녕하세요! CaseForMe에 오신 것을 환영합니다.
    먼저 맞춤형 케이스를 만들기 위해 몇 가지 여쭤볼게요.
    현재 어떤 분야에서 일하고 계신가요?

👤: IT 회사에서 마케팅 팀장을 하고 있어요.

🤖: IT 업계 마케팅 팀장이시군요!
    회사 규모는 어느 정도인가요?
    [스타트업(~50명)] [중소기업(50~300명)] [중견기업(300~1000명)] [대기업(1000명+)]

👤: [중견기업(300~1000명)]

🤖: 좋습니다. 경력은 몇 년 정도 되셨나요?

👤: 12년 정도요.

🤖: 12년 경력의 IT 중견기업 마케팅 팀장님이시군요.
    혹시 요즘 가장 고민되는 업무 이슈가 있으신가요? (없으시면 건너뛰어도 됩니다)

👤: AI 도구를 마케팅에 도입하려는데, 팀원들의 저항이 심해요.

🤖: 기술 도입과 조직 변화 관리에 대한 고민이시네요.
    프로필이 완성되었습니다! 이제 맞춤형 케이스를 추천해드릴게요.
```

### Phase 2: 케이스 선택

#### 케이스 카테고리 (6개 영역)
```
1. 전략적 의사결정
   - 시장 진입/철수, 사업 확장, M&A, 경쟁전략

2. 리더십과 조직관리
   - 조직 변화, 인재 관리, 갈등 해결, 문화 구축

3. 마케팅과 고객전략
   - 신제품 출시, 브랜드 전략, 가격 정책, 디지털 마케팅

4. 재무와 투자
   - 투자 의사결정, 자금 조달, 비용 구조, 수익 모델

5. 디지털 전환과 혁신
   - AI/DX 도입, 기술 전략, 플랫폼 비즈니스

6. 윤리와 사회적 책임
   - ESG, 이해관계자 갈등, 지속가능성
```

#### 난이도 설정
```
🟢 기본 (Basic): 명확한 문제, 충분한 데이터, 2가지 선택지
🟡 중급 (Intermediate): 복합적 문제, 부분적 데이터, 3~4가지 선택지
🔴 고급 (Advanced): 모호한 문제, 불완전한 데이터, 열린 결말, 윤리적 딜레마 포함
```

#### 챗봇 대화 예시
```
🤖: 프로필을 분석한 결과, 다음 케이스를 추천드립니다:

    ⭐ 추천 케이스: "AI 도구 도입과 팀 변화 관리"
    카테고리: 디지털 전환과 혁신 + 리더십
    난이도: 🟡 중급

    [이 케이스로 시작하기]  [다른 카테고리 보기]  [난이도 변경]
```

### Phase 3: 케이스 변환 및 제시

#### 변환 로직 (Personalization Engine)

원본 케이스를 사용자 프로필에 맞게 변환하는 핵심 로직:

```
변환 매핑:
├── 기업 → 사용자 업종의 유사 기업으로 변환
│   (예: "코카콜라 마케팅 전략" → "IT 중견기업의 B2B 마케팅 전략")
├── 인물 → 사용자 직급/역할에 맞는 인물로 변환
│   (예: "CEO가 직면한~" → "마케팅 팀장이 직면한~")
├── 규모 → 사용자 회사 규모에 맞게 조정
│   (예: "글로벌 100개국" → "국내 5개 지사")
├── 산업 맥락 → 사용자 산업의 트렌드/용어 적용
│   (예: "소비재 시장 포화" → "SaaS 시장 경쟁 심화")
└── 고민 연계 → 사용자가 입력한 실제 고민과 연결
    (예: "팀원 저항" 이슈를 케이스에 자연스럽게 포함)
```

#### 케이스 제시 형식
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 CASE: 넥스트소프트의 AI 마케팅 전환

배경:
넥스트소프트는 직원 450명 규모의 IT 중견기업으로,
B2B SaaS 솔루션을 제공하고 있다. 최근 3년간 매출
성장률이 15%에서 5%로 둔화되었다...

주인공:
김도현 마케팅 팀장 (경력 12년)은 AI 기반 마케팅
자동화 도구 도입을 추진하고 있다. 그러나...

문제 상황:
1. 팀원 8명 중 5명이 AI 도구 도입에 반대
2. 경쟁사는 이미 AI 마케팅을 도입하여 리드 전환율 30% 향상
3. 예산은 연간 2억원으로 제한되어 있음
4. CEO는 6개월 내 가시적 성과를 요구

📊 첨부 데이터:
- [표1] 3년간 마케팅 성과 추이
- [표2] 팀원 역량/성향 프로필
- [표3] AI 도구 비교 분석표
- [표4] 경쟁사 벤치마킹 데이터

❓ 핵심 질문:
당신이 김도현 팀장이라면, AI 마케팅 도구 도입을
어떻게 추진하겠습니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 케이스 읽기 UI
- 케이스 본문은 **접이식 섹션**으로 구분 (배경/문제/데이터)
- 데이터/차트는 **탭으로 전환**하여 확인
- **하이라이트 기능**: 중요한 부분 표시 가능
- 하단에 **"분석 시작하기"** 버튼

### Phase 4: 소크라테스식 토론 (핵심 대화 흐름)

#### 토론 구조 (5단계 대화)

```
Round 1: 상황 파악 (Understanding)
  "이 케이스에서 가장 핵심적인 문제는 무엇이라고 보시나요?"

Round 2: 분석 (Analysis)
  "팀원들이 반대하는 근본적인 이유는 무엇일까요?"
  "데이터를 보면 어떤 인사이트를 얻을 수 있나요?"

Round 3: 대안 탐색 (Alternatives)
  "이 상황에서 가능한 선택지를 3가지 이상 생각해보세요."
  "각 대안의 장단점은 무엇인가요?"

Round 4: 도전 질문 (Devil's Advocate)
  "만약 그 방법이 실패한다면 어떻게 되나요?"
  "반대 입장에서 생각해보면 어떤 논리가 가능할까요?"
  "당신이 간과하고 있는 리스크는 없을까요?"

Round 5: 의사결정 (Decision)
  "최종적으로 어떤 결정을 내리시겠습니까?"
  "그 결정의 근거를 3가지로 요약해주세요."
```

#### 챗봇 토론 대화 예시
```
🤖: 케이스를 다 읽으셨나요? 그럼 분석을 시작하겠습니다.
    먼저, 이 케이스에서 김도현 팀장이 직면한 가장 핵심적인
    문제는 무엇이라고 생각하시나요?

👤: AI 도구를 도입해야 하는데 팀원들이 반대하는 게 문제인 것 같아요.

🤖: 팀원들의 반대가 핵심 문제라고 보시는군요.
    그렇다면 한 가지 더 생각해보겠습니다.
    팀원들이 반대하는 "진짜" 이유는 무엇일까요?
    단순히 새로운 것에 대한 거부감일까요,
    아니면 더 깊은 이유가 있을까요?

👤: 아무래도 자기 일자리가 위협받을 수 있다는 불안감이 클 것 같아요.
    그리고 새로운 도구를 배워야 한다는 부담감도 있고요.

🤖: 좋은 분석입니다. 고용 불안과 학습 부담, 두 가지를
    짚어주셨네요.

    그런데 [표2]의 팀원 프로필을 보면, 반대하는 5명 중
    3명은 경력 15년 이상의 베테랑입니다.
    이 정보를 어떻게 해석하시나요?

👤: 아, 경력이 긴 분들일수록 기존 방식에 대한 자신감이 있어서
    변화를 더 거부할 수 있겠네요.

🤖: 그렇죠. 그렇다면 다른 각도에서 생각해봅시다.
    만약 AI 도구를 도입하지 않는다면 어떤 일이 벌어질까요?
    [표4]의 경쟁사 데이터를 참고해서 답해보세요.

👤: 경쟁사가 이미 30% 리드 전환율 향상을 보이고 있으니,
    도입하지 않으면 점점 뒤처질 거예요.

🤖: 맞습니다. 그렇다면 지금 가능한 선택지를 정리해봅시다.
    최소 3가지 다른 접근법을 제안해보시겠어요?

    💡 힌트: 꼭 "전면 도입 vs 미도입"의 이분법이 아닐 수 있습니다.

👤: 1. 전면 도입하고 교육 프로그램 운영
    2. 소규모 파일럿으로 시작해서 성과 증명 후 확대
    3. 찬성하는 팀원들만 먼저 도입하고 결과를 공유

🤖: 세 가지 좋은 대안을 제시하셨습니다.
    이제 각각에 대해 한번 도전해보겠습니다.

    2번 "소규모 파일럿"을 선택한다고 합시다.
    CEO가 6개월 내 가시적 성과를 요구하고 있는데,
    파일럿에 3개월을 쓰면 남은 3개월로 충분한 성과를
    보여줄 수 있을까요?

👤: 음... 그건 좀 빡빡하네요. 파일럿 기간을 2개월로
    줄이고 동시에 전체 팀 교육도 병행하면 어떨까요?

🤖: 현실적인 절충안이네요.
    그러면 최종 결정을 내려볼 시간입니다.

    당신이 김도현 팀장이라면, 어떤 결정을 내리겠습니까?
    그리고 그 결정의 근거를 3가지로 정리해주세요.
```

#### 토론 중 챗봇 행동 원칙
1. **정답을 말하지 않는다** - 소크라테스식 질문만 던짐
2. **데이터 활용을 유도** - 케이스 내 표/차트를 참조하게 함
3. **반대 관점을 제시** - Devil's Advocate 역할
4. **논리적 비약을 짚어줌** - "그 근거는 무엇인가요?"
5. **격려와 인정** - 좋은 분석에 대해 긍정적 피드백

```
■ 역량별 분석 (5개 영역)

1. 문제 정의력 ████████░░ 80/100
   - 핵심 문제를 정확히 파악함
   - 표면적 문제와 근본 원인을 구분함

2. 분석적 사고 ███████░░░ 70/100
   - 데이터 활용이 양호하나, [표3]의 비용 데이터를
     충분히 활용하지 못함

3. 창의적 대안 ████████░░ 80/100
   - 3가지 대안을 제시했으며, 절충안도 도출함
   - 더 혁신적인 접근(외부 전문가 활용 등)도 고려 가능

4. 리스크 관리 ██████░░░░ 60/100
   - 실행 리스크는 고려했으나, 기술적 리스크
     (도구 성능 문제)를 간과함

5. 실행 계획력 ████████░░ 80/100
   - 구체적 타임라인 제시
   - KPI 설정이 명확함
```

### 6.2 핵심 모듈

3. **Socratic Dialogue Engine**: 소크라테스식 질문 생성
4. **Assessment Engine**: 의사결정 분석 및 점수 산출
5. **Session Manager**: 대화 맥락 유지 및 진행 상태 관리

## 8. 예외 처리 시나리오 (검수 반영 보완)

### 8.1 맥락 이탈 대응
```
상황: 사용자가 케이스와 무관한 답변을 할 때

🤖: 좋은 의견이지만, 지금 다루고 있는 케이스로 다시 돌아가볼까요?
    김도현 팀장의 상황에서, [마지막 질문 반복]

재이탈 시:
🤖: 혹시 이 케이스가 어렵거나 흥미롭지 않으신가요?
    [다른 케이스 선택하기] [난이도 변경하기] [이어서 하기]
```

### 8.2 중간 저장 및 복귀
```
자동 저장:
- 사용자의 모든 답변은 실시간으로 세션에 저장
- 토론 진행 단계, 현재 라운드, 대화 히스토리 보존

복귀 시:
🤖: 안녕하세요! 지난번에 "넥스트소프트의 AI 전환" 케이스를
    Round 3까지 진행하셨네요. 이어서 하시겠어요?
    [이어서 하기] [처음부터 다시] [다른 케이스 선택]
```

### 8.3 토론 라운드 조절
```
토론 시작 시:
🤖: 분석을 시작하겠습니다. 토론 깊이를 선택해주세요.
    [⚡ 핵심만 빠르게 (3라운드, ~10분)]
    [📝 표준 분석 (5라운드, ~20분)]
    [🔬 심층 분석 (7라운드, ~35분)]
```

### 8.4 부적절 입력 대응
```
너무 짧은 답변:
🤖: 조금 더 구체적으로 설명해주시겠어요?
    어떤 근거에서 그렇게 생각하시나요?

공격적/부적절 입력:
🤖: 저는 경영 케이스 토론에 집중하고 있어요.
    케이스에 대한 의견을 나눠볼까요?

"모르겠다" 류의 답변:
🤖: 괜찮습니다! 힌트를 드릴게요.
```

## 9. 원본 케이스 저작권 참고

MBA 케이스 원본은 Harvard Business Publishing 등의 저작권이 있으므로, 다음 방식으로 처리:

1. **원본 케이스를 직접 사용하지 않음** - 구조와 교훈만 참고
2. **자체 케이스 시나리오 창작** - 유사한 의사결정 구조를 가진 독창적 시나리오 제작
3. **변환 시 원본과 충분히 다른 컨텍스트 생성** - 업종, 인물, 상황을 완전히 변경
4. 필요 시 Harvard Business Publishing 라이선스 검토
