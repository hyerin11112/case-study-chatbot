# 기획서 검수 리포트

## 검수 기준
1. **완성도** - 기획에 필요한 모든 요소가 포함되었는가?
2. **실현 가능성** - 기술적으로 구현 가능한가?
3. **교육 효과** - MBA 케이스 스터디의 교육적 가치를 잘 살렸는가?
4. **사용자 경험** - 사용자 입장에서 자연스럽고 효과적인가?
5. **차별성** - 기존 도구 대비 명확한 장점이 있는가?

- [x] 전체 사용자 흐름 (6단계)

- [x] 개발 로드맵
- [x] 저작권 고려사항

### 보완 필요 항목 ⚠️
- [ ] **에러 처리 시나리오**: 사용자가 맥락과 무관한 답변을 할 때의 대응

## 2. 실현 가능성 평가: ★★★★☆ (4/5)

### 기술적으로 구현 가능한 부분 ✅
- **프로필 기반 케이스 변환**: LLM(Claude API)의 프롬프트 엔지니어링으로 충분히 가능
- **소크라테스식 대화**: 시스템 프롬프트 + 대화 컨텍스트 관리로 구현 가능
- **결과 리포트 생성**: LLM 기반 분석 + 규칙 기반 점수 산출 조합 가능
- **채팅 UI**: 기존 채팅 프레임워크(예: React + WebSocket) 활용 가능

### 기술적 도전 과제 ⚠️
| 도전 과제 | 난이도 | 해결 방안 |
|----------|--------|----------|
| 케이스 변환 품질 일관성 | 높음 | 변환 템플릿 + few-shot 프롬프트 + 검증 파이프라인 |
| 소크라테스 대화 자연스러움 | 중간 | 대화 단계별 프롬프트 분리 + 컨텍스트 윈도우 관리 |
| 평가 점수 공정성 | 높음 | 루브릭 기반 평가 기준 명문화 + 다중 LLM 교차 검증 |
| 데이터 표/차트 동적 생성 | 중간 | 템플릿 기반 데이터 생성 + Chart.js 연동 |
| 토론 맥락 장기 유지 | 중간 | 대화 요약 + 핵심 포인트 추출 방식으로 컨텍스트 압축 |

### 검수 의견
> MVP 수준에서는 충분히 실현 가능. 다만 케이스 변환 품질과 평가 점수 공정성은 초기에 사람의 검수가 필요할 수 있음. 점진적 개선(human-in-the-loop) 접근을 권장함.

## 3. 교육 효과 평가: ★★★★★ (5/5)

### HBS 케이스 메서드 충실도
| HBS 원칙 | 기획서 반영 여부 | 평가 |
|----------|----------------|------|
| 실제 기업 기반 시나리오 | ✅ 실제 케이스 구조 참고 + 맞춤 변환 | 우수 |
| 불완전한 정보에서 의사결정 | ✅ 난이도별 정보 제공량 차등 | 우수 |
| 소크라테스식 질문법 | ✅ 5단계 토론 구조 설계 | 우수 |
| 교수의 중립적 역할 | ✅ "정답을 말하지 않는다" 원칙 명시 | 우수 |
| 다양한 관점 노출 | ✅ Devil's Advocate 역할 설계 | 우수 |
| 성찰과 메타인지 | ✅ 결과 리포트에서 사고과정 분석 | 우수 |

### 추가 교육적 가치
- **개인화**: 자신의 업무 맥락에서 케이스를 경험하므로 전이 학습 효과 극대화
- **즉각 피드백**: 실제 MBA 수업에서는 없는 개인별 상세 피드백 제공
- **반복 학습**: 동일 케이스를 다른 관점에서 재도전 가능
- **심리적 안전감**: AI 상대이므로 틀려도 부담 없이 시도 가능

### 검수 의견
> 교육 효과 측면에서 가장 강점이 두드러짐. 특히 "내 상황에 맞는 케이스"라는 개인화 요소가 학습 동기와 전이 효과를 크게 높일 것으로 기대됨.

## 4. 사용자 경험 평가: ★★★★☆ (4/5)

### 잘 설계된 부분 ✅
- 대화형 온보딩으로 자연스러운 프로필 수집
- 퀵 리플라이 버튼으로 입력 부담 최소화
- 접이식 케이스 뷰어로 정보 과부하 방지
- 단계별 진행 표시로 현재 위치 인식

### UX 개선 제안 ⚠️

1. **토론 길이 조절 옵션 필요**
   - 현재: 고정 5라운드
   - 개선: 사용자가 "빠른 분석(3라운드)" / "심층 분석(7라운드)" 선택 가능

2. **중간 저장 기능**
   - 긴 토론 중 이탈 시 세션을 저장하고 이어서 진행할 수 있어야 함

3. **힌트 시스템 구체화**
   - 현재: 하단에 [힌트] 버튼만 표시
   - 개선: 단계별로 다른 힌트 제공 (분석 프레임워크 제안, 데이터 포인트 하이라이트 등)

4. **모바일 UX 강화**
   - 케이스 본문이 긴 경우 모바일에서의 가독성 확보 방안 필요
   - 음성 입력 지원 고려

### 검수 의견
> 기본적인 UX 흐름은 잘 설계됨. 토론 길이 조절과 중간 저장 기능은 MVP에서도 반드시 포함할 것을 권장함.
