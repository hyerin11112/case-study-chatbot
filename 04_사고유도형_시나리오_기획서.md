# MBA 케이스 스터디 챗봇 시나리오 (사고 유도형)

## 핵심 철학: "답을 찾게 하지 말고, 질문을 만들게 하라"

기존 시나리오가 **"케이스를 읽고 → 질문에 답하고 → 평가받는"** 구조였다면,
사고 유도형은 **"스스로 문제를 정의하고 → 스스로 질문을 던지고 → 스스로 판단 기준을 세우는"** 구조다.

> 핵심 차이: AI가 질문하는 것이 아니라, **사용자가 질문을 만들어내도록** 유도한다.

---

## 전체 흐름도

```
[1. 온보딩]        →  [2. 상황 투입]      →  [3. 자기 진단]
프로필 입력             불완전한 상황만           "뭐가 문제라고
                       던져줌 (질문 없음)         생각하시나요?"
                                                      ↓
[6. 메타인지 성찰]  ←  [5. 갈림길 선택]   ←  [4. 정보 탐색]
"왜 그렇게              트레이드오프 직면          사용자가 필요한
 생각했나요?"           + 되돌릴 수 없는 선택      정보를 직접 요청
```

---

## Phase 1: 온보딩 (기존과 동일)
사용자 프로필 수집 → 맞춤형 케이스 변환의 기반

---

## Phase 2: 상황 투입 - "질문 없이 던진다"

### 핵심 원칙
- **핵심 질문(❓)을 제시하지 않는다**
- 상황만 주고, 사용자가 스스로 "이게 문제다"를 정의하게 한다
- 정보를 일부러 빠뜨려서 "뭔가 부족하다"는 감각을 유발한다

### 대화 예시
```
🤖: 상황을 하나 드리겠습니다. 읽어보세요.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
넥스트소프트 마케팅팀 주간회의록 (발췌)

김도현 팀장: "이번 분기 리드 전환율이 또 떨어졌습니다.
경쟁사 B사는 올해 들어 전환율이 30% 올랐다는데..."

박수진 선임: "B사가 뭘 다르게 하는 건지 모르겠어요."

이재훈 대리: "AI 도구 얘기가 나올 것 같은데, 전 반대예요.
지금 방식이 잘못된 게 아니라 예산이 부족한 거예요."

최민경 과장: (침묵)

김도현 팀장: "CEO가 다음 달 경영회의에서 마케팅 혁신
방안을 보고하라고 합니다. 시간이 없어요."

(회의록 끝)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🤖: 읽으셨으면 말씀해주세요.

👤: 네, 읽었어요.

🤖: 어떤 생각이 드시나요?
```

### 기존 시나리오와의 차이
| 항목 | 기존 | 사고 유도형 |
|------|------|-----------|
| 제시 형식 | 정리된 케이스 문서 (배경/문제/데이터/질문) | 가공되지 않은 회의록, 이메일, 보고서 단편 |
| 핵심 질문 | "당신이라면 어떻게 하겠습니까?" 명시 | 질문 없음. "어떤 생각이 드시나요?"만 |
| 데이터 | 표/차트를 처음부터 제공 | 사용자가 요청해야 제공 |
| 문제 정의 | AI가 문제를 정의해줌 | 사용자가 직접 문제를 정의해야 함 |

---

## Phase 3: 자기 진단 - "문제를 정의하게 한다"

### 핵심 원칙
- 사용자가 말한 것을 **그대로 되물어** 사고를 깊게 만든다
- "왜?"를 직접 묻지 않고, **사용자의 말 속 전제를 드러낸다**
- 사용자가 자신의 사고 패턴을 인식하게 만든다

### 대화 예시
```
👤: AI 도구를 도입해야 할 것 같아요. 경쟁사도 하고 있으니까.

🤖: "경쟁사가 하고 있으니까"라고 하셨는데,
    경쟁사가 하지 않았다면 도입할 필요가 없을까요?

👤: 음... 그건 아닌데, 전환율이 떨어지고 있으니까요.

🤖: 전환율이 떨어지는 원인이 AI 도구가 없어서라고
    보시는 건가요?

👤: 확실하진 않아요. 다른 이유일 수도 있죠.

🤖: 그렇다면 지금 확실히 알고 있는 것과,
    모르는 것을 구분해보시겠어요?

👤: 알고 있는 건... 전환율이 떨어지고 있다는 것,
    경쟁사 B사의 전환율이 올랐다는 것.
    모르는 건... 왜 떨어지는지, B사가 정확히 뭘 했는지.

🤖: 좋습니다. 그러면 지금 의사결정을 내리기에
    부족한 정보가 있다는 뜻인가요?

👤: 네, 더 알아봐야 할 것 같아요.

🤖: 무엇을 알아보고 싶으세요?
```

### 기존 시나리오와의 차이
| 항목 | 기존 | 사고 유도형 |
|------|------|-----------|
| AI의 역할 | 구조화된 질문을 순서대로 던짐 | 사용자의 말을 거울처럼 비춰줌 |
| 진행 방식 | Round 1→2→3→4→5 고정 순서 | 사용자의 사고 흐름에 따라 유동적 |
| 핵심 기법 | 소크라테스식 질문 (AI가 질문) | 리플렉션 기법 (사용자가 질문을 만듦) |
| 정보 제공 | 모든 데이터를 미리 제공 | 사용자가 요청한 것만 제공 |

---

## Phase 4: 정보 탐색 - "사용자가 직접 파고든다"

### 핵심 원칙
- 데이터를 **사용자가 요청해야만** 제공한다
- 어떤 정보를 요청하느냐 자체가 사고력의 척도
- 요청하지 않은 중요 정보는 끝까지 주지 않는다 (나중에 성찰에서 활용)

### 대화 예시
```
👤: B사가 정확히 뭘 했는지 알 수 있나요?

🤖: B사 관련 업계 기사가 하나 있습니다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[IT Daily] B사, AI 마케팅 자동화 도입 6개월 성과 발표
- 리드 전환율 30% 향상
- 마케팅팀 인원 변동 없음
- 도입 초기 3개월간 성과 하락 후 반등
- 투자 비용: 연 3억원
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🤖: 더 알아보고 싶은 것이 있으신가요?

👤: 우리 팀 예산이 얼마인지 알 수 있나요?

🤖: 넥스트소프트 마케팅팀 연간 예산은 2억원입니다.
    현재 집행률은 68%입니다.

👤: 팀원들 현황도 알고 싶어요.

🤖: 마케팅팀 인원 현황입니다.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
총 8명
- AI/디지털 도구 도입 찬성: 3명 (경력 3~7년)
- 반대: 4명 (경력 10~18년)
- 의견 미표명: 1명 (최민경 과장, 경력 9년)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🤖: 또 필요한 정보가 있으신가요,
    아니면 충분히 파악하셨나요?

👤: 이 정도면 될 것 같아요.

🤖: 그런데 한 가지 여쭤볼게요.
    회의록에서 최민경 과장이 침묵하고 있었는데,
    그 부분은 신경 쓰이지 않으셨나요?

👤: 아... 그걸 놓쳤네요. 왜 침묵했을까요?

🤖: 제가 답을 드리기보다,
    팀장인 당신이라면 어떻게 알아보시겠어요?
```

### 정보 제공 규칙
```
Tier 1 (요청 시 즉시 제공):
  - 재무 데이터, 시장 데이터, 팀 구성

Tier 2 (요청 시 제공하되 불완전하게):
  - 경쟁사 정보 (일부만 공개, 나머지는 "확인 불가")
  - 고객 데이터 (샘플만 제공)

Tier 3 (요청하지 않으면 절대 제공 안 함):
  - 숨겨진 맥락 (최민경 과장의 사정, CEO의 숨은 의도 등)
  - 이 정보를 요청했는지 여부가 성찰 리포트에 반영됨
```

## Phase 5: 갈림길 선택 - "되돌릴 수 없는 결정"

### 핵심 원칙
- 단순히 "최종 결정"을 내리는 게 아니라, **중간중간 돌이킬 수 없는 선택**을 하게 함
- 선택의 결과가 즉시 드러나고, 새로운 상황이 전개됨
- **트레이드오프**를 명확히 체감하게 함

### 대화 예시
```
🤖: 지금까지 파악한 것을 바탕으로,
    내일 아침 팀 회의에서 첫 마디를 해야 합니다.
    어떻게 시작하시겠어요?

    단, 이 발언은 되돌릴 수 없습니다.
    팀원들은 이 첫 마디로 당신의 방향을 판단합니다.

👤: "여러분, 저도 아직 답을 정하지 못했습니다.
    함께 고민해봤으면 합니다."

🤖: 당신은 솔직하게 시작했습니다.

    그런데 이재훈 대리가 바로 반응합니다:
    "팀장님이 답도 없이 회의를 여신 건가요?
    CEO 보고가 한 달 남았는데요."

    팀의 분위기가 약간 동요합니다.
    어떻게 대응하시겠어요?

👤: "답이 없는 게 아니라, 여러분의 의견 없이
    제가 혼자 결정하고 싶지 않은 겁니다."

🤖: 박수진 선임이 고개를 끄덕입니다.
    하지만 이재훈 대리의 표정은 여전히 회의적입니다.

    이 시점에서 선택이 필요합니다:

    A) 찬성파 3명과 먼저 파일럿 팀을 꾸린다
       → 반대파는 소외감을 느낄 수 있음
    B) 전체 팀 워크숍을 열어 합의를 만든다
       → 시간이 2주 소요, CEO 보고 일정 압박
    C) 이재훈 대리와 1:1 면담을 먼저 한다
       → 핵심 반대자를 설득하면 나머지는 따라올 수 있음

    어떤 것을 선택하시겠어요?
    혹은 다른 방법이 있다면 말씀해주세요.

👤: C를 선택할게요.

🤖: 이재훈 대리와 1:1 면담 자리입니다.
    대리가 먼저 말합니다:

    "팀장님, 솔직히 말할게요.
    AI 도구 들어오면 제가 10년 해온 일이
    쓸모없어지는 거 아닌가요?"

    이 말에 어떻게 답하시겠어요?
```

### 기존 시나리오와의 차이
| 항목 | 기존 | 사고 유도형 |
|------|------|-----------|
| 의사결정 | 마지막에 한 번 최종 결정 | 중간중간 연쇄적 선택, 결과가 즉시 전개 |
| 결과 | 토론 끝나고 리포트로 평가 | 선택 → 즉각적 상황 변화 → 다시 선택 |
| 느낌 | 시험/평가 느낌 | 시뮬레이션/롤플레이 느낌 |
| 긴장감 | 낮음 (언제든 수정 가능) | 높음 (되돌릴 수 없음) |
| 트레이드오프 | 이론적으로 설명 | 직접 체감 |

## Phase 6: 메타인지 성찰 - "당신의 사고 패턴을 보여드립니다"

### 핵심 원칙
- 점수를 매기지 않는다
- 대신 **"당신은 이렇게 생각하는 경향이 있습니다"**를 보여준다
- 놓친 관점, 건너뛴 정보, 무의식적 전제를 드러낸다

### 대화 예시
```
🤖: 케이스가 끝났습니다.
    점수를 드리기보다, 당신의 사고 과정을
    함께 되돌아보겠습니다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🪞 당신의 사고 패턴 분석

■ 문제 정의 방식
당신은 처음에 "AI 도구 도입"을 문제로 정의했습니다.
하지만 실제 문제는 "전환율 하락의 원인 파악"이었을 수 있습니다.
→ 해결책을 먼저 떠올리고, 문제를 거기에 맞추는 경향
   (이것을 "Solution-first bias"라고 합니다)

■ 정보 탐색 패턴
요청한 정보: 경쟁사 동향, 예산, 팀원 현황 (3건)
요청하지 않은 정보:
  ❌ 전환율 하락의 구체적 원인 분석 데이터
  ❌ 기존 마케팅 프로세스의 병목 지점
  ❌ 고객 이탈 패턴
→ "왜 떨어졌는가"보다 "어떻게 올릴 것인가"에
   먼저 집중하는 경향

■ 의사결정 스타일
최민경 과장의 침묵을 처음에 놓쳤습니다.
→ 명시적으로 드러난 갈등(이재훈 대리)에 집중하고,
   드러나지 않은 신호를 간과하는 경향

이재훈 대리와 1:1 면담을 선택했습니다.
→ 전체 합의보다 핵심 인물 설득을 선호하는 스타일
   (장점: 효율적 / 리스크: 나머지 팀원 소외 가능)

■ 당신이 무의식적으로 전제한 것들
1. "AI 도구를 도입하면 전환율이 오를 것이다"
   → 검증되지 않은 가정
2. "반대하는 사람을 설득해야 한다"
   → "반대 의견이 옳을 수 있다"는 가능성은 고려하지 않음
3. "CEO 보고 일정은 변경할 수 없다"
   → 정말 그런지 확인하지 않음
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🤖: 이 중에서 가장 의외였던 부분이 있으신가요?

👤: "반대 의견이 옳을 수 있다"는 걸 정말 생각 안 했네요.

🤖: 흥미롭네요. 만약 이재훈 대리의 말이 맞다면,
    즉 전환율 하락이 AI 부재 때문이 아니라
    예산 부족 때문이라면,
    당신의 결정은 어떻게 달라졌을까요?

👤: 완전히 다른 접근을 했을 것 같아요...

🤖: 이것이 이 케이스의 핵심입니다.
    "내가 전제한 것이 틀렸을 가능성"을 항상 점검하는 것.

    다음에 실제 업무에서 의사결정을 내릴 때,
    "내가 지금 당연하다고 여기는 것은 무엇인가?"를
    먼저 물어보시면 좋겠습니다.
```

### 기존 리포트와의 차이
| 항목 | 기존 리포트 | 사고 유도형 성찰 |
|------|-----------|----------------|
| 형식 | 점수 (78/100) + 잘한 점/개선점 | 사고 패턴 분석 + 대화형 성찰 |
| 초점 | "무엇을 잘했고 못했나" (평가) | "왜 그렇게 생각했나" (인식) |
| 느낌 | 시험 채점 | 코칭/상담 |
| 사용자 반응 | "아, 이건 못했구나" | "아, 나는 이렇게 생각하는 경향이 있구나" |
| 학습 전이 | 이 케이스에서의 교훈 | 모든 의사결정에 적용 가능한 메타인지 |

## 전체 비교 요약

### 구조 비교

```
[기존 시나리오]
온보딩 → 케이스 선택 → 케이스 읽기 → AI가 질문 → 답변 → 점수

사용자 역할: 응답자 (질문에 대답하는 사람)
AI 역할: 시험관 (질문하고 평가하는 사람)
핵심 경험: "잘 답변해야 한다"

[사고 유도형 시나리오]
온보딩 → 상황 투입 → 스스로 문제 정의 → 스스로 정보 탐색
→ 연쇄적 선택 → 사고 패턴 성찰

사용자 역할: 당사자 (상황 속 의사결정자)
AI 역할: 거울 (사용자의 사고를 비춰주는 존재)
핵심 경험: "내가 어떻게 생각하는지 알게 되었다"
```

### 항목별 상세 비교

| 비교 항목 | 기존 시나리오 | 사고 유도형 |
|----------|-------------|-----------|
| **케이스 제시** | 정리된 문서 (배경/문제/데이터/질문) | 가공 안 된 원본 (회의록, 이메일, 메모) |
| **핵심 질문** | AI가 명시적으로 제시 | 없음. 사용자가 스스로 만들어야 함 |
| **데이터 제공** | 처음부터 전부 제공 | 사용자가 요청한 것만 제공 |
| **대화 구조** | 5라운드 고정 순서 | 사용자 사고 흐름에 따라 유동적 |
| **AI 질문 방식** | 소크라테스식 (구조화된 질문) | 리플렉션 (사용자의 말을 되비춤) |
| **의사결정** | 마지막에 1회 최종 결정 | 중간중간 되돌릴 수 없는 연쇄 선택 |
| **선택 결과** | 리포트에서 사후 평가 | 즉각적 상황 변화로 체감 |
| **결과물** | 점수 (78/100) + 평가 | 사고 패턴 분석 + 무의식적 전제 드러냄 |
| **학습 효과** | "이 케이스에서 배운 것" | "나의 사고 습관을 인식한 것" |
| **감정** | 시험 → 채점 (긴장→안도) | 몰입 → 발견 (호기심→통찰) |
| **재사용 가치** | 같은 케이스 반복 시 효과 감소 | 같은 케이스도 다른 선택지로 새로운 경험 |
| **실무 전이** | 중간 (케이스별 교훈) | 높음 (사고 패턴 자체를 개선) |
