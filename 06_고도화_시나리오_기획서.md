# CaseForMe 고도화 시나리오 기획서
## MBA 교수법 분석 기반 Q1-Q4 구조 개선

## 1. 고도화 배경

### 1-1. MBA 교안 분석 결과

두 MBA 실제 교안(전경련 비즈니스 스쿨 사례연구, D사 MBA 전략경영 과제)에서 공통 교수법 패턴을 발견했다.

**문서1 — 전경련 인사조직 사례연구 (박찬균 교수)**
- Case 1 (김 행장): 사실파악 → 이해관계자 감정/관점 → "당신이라면" 의사결정
- Case 2 (이 팀장): 4명의 팀원 각각 상황이 다른 다중 이해관계자 구조
- Case 3 (콜린): 관계 비교분석 → 조직 관점 평가 → 개선안 제시

**문서2 — MBA 전략경영 5 Forces 과제**
- 프레임워크(5 Forces) 기반 구조화된 분석
- 각 항목에 근거(evidence) 필수 제시
- 종합 평가로 수렴

**공통 패턴**: 사실 이해 → 다관점 분석 → 자기 투영 의사결정 → 비판적 검증

### 1-2. 현재 Q1-Q3의 핵심 문제점

| 문제 | 현재 | MBA 교안 |
|------|------|----------|
| 이해관계자 분석 없음 | Q1이 바로 "핵심 문제" 질문 | 먼저 각 이해관계자 입장을 파악 |
| 관점 전환 없음 | 사용자 관점만 고려 | "직원 입장에서 어떤 기분?" 같은 공감 질문 |
| 케이스 깊이 부족 | 1명의 주인공 + 숫자 데이터 | 3-4명의 구체적 인물, 각각 다른 이해관계 |
| 실행계획 부재 | 결정 + 근거만 요구 | "어떻게 변화를 이끌어가겠는가" (구체적 실행) |
| 프레임워크 연결 없음 | 없음 | LMX 이론, 변화관리, 5 Forces 등 연결 |

## 2. 개선 시나리오: Q1-Q4 구조

### 2-1. 새로운 질문 프레임워크

```
Q1: 이해관계자 진단 (Stakeholder Diagnosis)
    질문 패턴: "이 사례의 핵심 이해관계자는 누구이며, 각각의 입장은?"
    MBA 교안 매핑: 사실 파악 + 관점 전환
    개발 역량: 다관점 분석, 공감 능력

Q2: 핵심 딜레마 정의 (Core Tension)
    질문 패턴: "이 상황의 핵심 딜레마는 무엇이고, 왜 쉽게 해결되지 않는가?"
    MBA 교안 매핑: "어떤 점이 문제라고 생각하는가?"
    개발 역량: 문제 구조화, 딜레마 인식

Q3: 의사결정 + 실행 계획 (Decision + Implementation)
    질문 패턴: "당신이 [주인공]이라면 어떤 결정을 내리고,
               어떤 순서로 실행하겠는가? Q1의 이해관계자 반응을 고려하라."
    MBA 교안 매핑: "자신이 김 행장의 뒤를 이어 경영하게 된다면"
    개발 역량: 의사결정, 실행 설계, 이해관계 조율

Q4: 리스크 평가 + 대안 검증 (Risk + Devil's Advocate)
    질문 패턴: "계획이 실패한다면 가장 큰 원인은?
               만약 정반대 결정을 했다면 장단점은?"
    MBA 교안 매핑: "A회사에서 콜린의 리더십을 어떻게 평가하겠는가?"
    개발 역량: 메타인지, 대안 사고, 비판적 자기 검증
```

### 2-2. 케이스 생성 강화

기존: 단일 주인공 + 데이터 중심
변경: 3-4명의 이해관계자 구조 + 관계 역학 + 핵심 사건

케이스 필수 요소:
1. **배경 설정** (3-4문장) — 업종, 회사 규모, 현재 상황
2. **주인공** (2-3문장) — 사용자와 유사한 직급/경력
3. **이해관계자 3-4명** — 각각 이름, 역할, 입장이 다름
   - 최소 2명은 정당하지만 상충하는 이해관계를 가짐
4. **핵심 사건** (2-3문장) — 의사결정을 촉발하는 구체적 사건
5. **참고 데이터** (3-4개) — 최소 1개는 해석이 갈리는 모호한 데이터
6. **제약 조건** (2-3개) — 시간, 자원, 관계적 제약

### 2-3. 후속 질문 패턴 (인정-도전-연결)

기존: "사고를 더 깊게 만드는 후속 질문"
변경: MBA 교수의 인정-도전-연결 패턴

1. 사용자가 잘 파악한 점을 구체적으로 인정 (1문장)
2. 놓친 관점이나 전제를 부드럽게 도전 (1문장)
3. 다음 질문으로의 연결 (필요시)

### 2-4. 인사이트 리포트 5섹션 구조

기존: 고려 포인트 + 키포인트 + 토론 가이드 (3섹션)
변경: 5개 섹션으로 확장

```
[1. 분석 요약] — 사용자 답변을 체계적으로 정리
[2. 고려할 포인트] — 잘한 점 + 놓친 관점 (근거 포함)
[3. 관련 경영 프레임워크] — 이 사례에 적용 가능한 이론 1개 소개
    → 변화관리(Kotter 8단계), LMX 이론, 양면성 리더십 등
    → 프레임워크를 사례에 적용한 간단한 예시
[4. 핵심 키포인트] — 사용자 직무에 맞춘 3가지 교훈 + 실무 액션
[5. 실무 적용 질문] — "현재 직장에서 유사한 상황이 있다면?"
    → 케이스 학습을 실제 업무로 연결하는 브릿지
```

## 4. 검증 기준

1. **MBA 교수법 정합성**: Q1-Q4가 사실파악→관점전환→의사결정→메타인지 순서인가?
2. **케이스 품질**: 이해관계자가 3명 이상이고, 정답이 없는 딜레마인가?
3. **인사이트 가치**: 사용자가 "이걸 내 업무에서 어떻게 쓰지?"에 답을 얻는가?
4. **프레임워크 연결**: 경영 이론을 통해 전이 가능한 지식이 생기는가?
